# Answer Key with Explanations

## Question 1: **A, B, D**
Target audience sections include demographics (age range), platform preferences, and expected skill levels. Shader settings and bandwidth are technical specs.

## Question 2: **A, B**
Physics response requires a Rigidbody for physics simulation and a Collider for physical interaction. Other components serve different purposes.

## Question 3: **E (All of the above)**
Blueprints support Integer, Float, Vector, Actor Reference, and many other data types as variables.

## Question 4: **A, B, D**
Vertex shaders process vertices, can modify positions, and handle transformations. They execute before fragment shaders, which determine pixel color.

## Question 5: **A, B, C, E**
Particle count, texture resolution, overdraw, and collision complexity significantly impact performance. Emitter names have no performance effect.

## Question 6: **A, B, D**
Bind pose is the default reference pose (T-pose/A-pose) where skin weights are assigned and serves as the skeleton's reference.

## Question 7: **A, B, D**
Point-E, Shap-E, and Meshy AI generate 3D models. ChatGPT is text-based, and Grammarly is for writing assistance.

## Question 8: **A, B, D**
NavMesh provides pathfinding, defines walkable surfaces, and enables obstacle avoidance. It's not related to shaders or audio.

## Question 9: **A, B, C, D**
Additive, Alpha Blend, Multiplicative, and Subtractive are all valid blend modes. Raycast is a physics operation.

## Question 10: **A, B, D**
Base Color, Normal, and Roughness control surface appearance. World Position and Get Actor Location are utility nodes.

## Question 11: **A, B, D, E**
Monetization covers purchases, ads, pricing models, and mechanics like loot boxes. Vertex buffers are technical implementation details.

## Question 12: **A, B, D**
Awake(), Start(), and OnEnable() run during initialization. Update() runs every frame, and OnDestroy() runs during destruction.

## Question 13: **A, B, C, E**
Sequencer handles cinematics, camera animation, actor manipulation, and audio sync. Shader editing uses the Material Editor.

## Question 14: **A, B, D**
Subsurface scattering, dual-lobe specular, and translucency create realistic skin. Flat shading and wireframe are not suitable for realistic rendering.

## Question 15: **A, B, C**
Rate over Time, Rate over Distance, and Burst emission control particle emission. LOD and audio pitch are unrelated.

## Question 16: **A, B, D**
IK, ragdoll physics, and runtime bone manipulation are procedural. Keyframe animation and sprite cycling are predefined.

## Question 17: **A, B, D, E**
AI can generate dialogue, storylines, backstories, and branching narratives. Compile-time optimization is technical, not narrative.

## Question 18: **A, B, D, E**
Cinemachine provides intelligent cameras, blending, follow/look-at systems, and procedural shake. Network replication is separate.

## Question 19: **A, B, C, D**
Functions, Macros, Event Dispatchers, and Blueprint Interfaces enable reusability. Level Streaming is for world management.

## Question 20: **A, B, D, E**
Bloom, AO, color grading, and motion blur are post-processing effects. Mesh decimation is an optimization technique.

## Question 21: **A, B, C, E**
Scope includes features, timeline, platforms, and content boundaries. Bug tracking is development process, not scope.

## Question 22: **A, B, D**
ScriptableObjects are data containers, enable asset-based config, and reduce memory for shared data. They don't handle networking or physics.

## Question 23: **A, B, D**
Construction Script runs in editor, sets up components, and enables procedural generation. It complements, not replaces, Event Graph.

## Question 24: **A, B, C, D, E (All correct)**
All five are valid shader types in modern rendering pipelines: Vertex, Fragment/Pixel, Geometry, Compute, and Tessellation.

## Question 25: **A, B, D**
GPU particles benefit from parallel processing, reduce CPU load, and support higher counts. Audio integration and auto-LOD aren't GPU particle features.

## Question 26: **A, B, C, E**
Lerp, additive blending, layered animation, and cross-fades are animation blending techniques. Texture filtering is for images.

## Question 27: **A, B, D, E**
Stable Diffusion generates concept art, textures, style transfers, and UI designs. It cannot simulate physics.

## Question 28: **A, B, D, E**
The new Input System offers action mapping, multi-device support, runtime rebinding, and input buffering. Shaders are unrelated.

## Question 29: **A, B, C, D**
Delay, Set Timer, Timeline, and Get World Delta Seconds handle timing. Spawn Actor creates objects, not timing.

## Question 30: **A, B, D**
Normal maps add detail without geometry, store directions in RGB, and affect lighting. They don't increase polygons or require tessellation.

## Question 31: **A, B, D, E**
Progression includes XP systems, skill trees, unlockables, and difficulty scaling. Compiler flags are technical implementation.

## Question 32: **A, B, C, D**
GetComponent, events/delegates, static variables, and ScriptableObject events enable script communication. Mesh manipulation isn't communication.

## Question 33: **A, B, D, E**
GAS handles ability activation, cooldowns, effects, and attributes. Texture streaming is a rendering system.

## Question 34: **A, B, D, E (4 correct)**
Energy conservation, Fresnel, microfacet theory, and metallic workflow are PBR concepts. Sprite batching is a 2D optimization.

## Question 35: **A, B, D, E**
Niagara offers stack modularity, GPU simulation, data interfaces, and visual scripting. Audio integration isn't a Niagara advantage.

## Question 36: **A, C, D**
Root motion is animation-driven movement with accurate foot placement and better sync. It doesn't mean independent controller movement.

## Question 37: **A, B, D, E**
DALL-E generates concept art, environment sketches, prop designs, and UI mockups. It cannot optimize C++ code.

## Question 38: **A, B, C, E**
Sprite Renderer, Tilemap, 2D Physics, and 2D Animation support 2D development. Terrain system is for 3D environments.

## Question 39: **A, C, D**
Execution pins control order, sequential flow, and branching. Data pins pass data, not execution pins.

## Question 40: **A, B, D, E**
Displacement shaders modify actual vertices, create real geometry, work best with tessellation, and use heightmaps. They don't just affect normals.

## Question 41: **A, B, D, E**
Fire uses sprite sheets, emissive materials, upward velocity, and color gradients. Audio analysis doesn't create fire effects.

## Question 42: **A, B, D, E**
Animation Notifies trigger sounds, spawn effects, fire gameplay events, and sync footsteps. They don't modify skeletons.

## Question 43: **A, B, D, E**
AI assists with layout generation, difficulty balancing, enemy placement, and playtest analysis. Hardware drivers are system-level.

## Question 44: **A, B, D, E**
Canvas, RectTransform, EventSystem, and Graphic Raycaster are UI components. MeshRenderer is for 3D rendering.

## Question 45: **A, B, D, E**
Data Tables provide spreadsheet storage, CSV import, structured data, and designer-friendly editing. Shaders are generated elsewhere.

## Question 46: **A, B, C, D, E (All correct)**
All five are valid shader optimizations: reducing samples, simpler math, fewer branches, LOD switching, and lookup tables.

## Question 47: **A, B, D, E**
VFX Graph offers visual scripting, GPU acceleration, high counts, and compute shader integration. It doesn't synthesize audio.

## Question 48: **A, B, D, E**
Keyframe reduction, curve simplification, quantization, and removing redundant channels compress animations. Adding bones increases size.

## Question 49: **A, B, D, E**
Midjourney generates marketing art, mood boards, style references, and character portraits. It cannot write network code.

## Question 50: **A, B, D, E**
Addressables provide async loading, memory management, remote hosting, and dependency tracking. Code compilation is separate.
